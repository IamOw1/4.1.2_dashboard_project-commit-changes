## 🚀 Быстрый старт

### Требования

- Python 3.8+
- Установленные зависимости из `requirements.txt`
- Запущенный API сервер

### Запуск HTML интерфейса

```bash
# 1. Откройте веб-браузер
# 2. Перейдите на http://localhost:8080 (или по адресу вашего web-сервера)
# 3. Нажмите кнопку "Connect" для подключения к API
```

### Запуск Streamlit приложения

```bash
# В корневой папке проекта выполните:
streamlit run dashboard/app.py

# Приложение откроется в браузере на http://localhost:8501
```

## 🖥️ HTML интерфейс

### Возможности

- 🎨 **Стильный темный дизайн** - комфортный для глаз интерфейс
- ⚡ **Быстрая отзывчивость** - работает на всех современных браузерах
- 📱 **Адаптивный дизайн** - отлично выглядит на мобильных устройствах

### Основные компоненты

#### 1. **Панель подключения**
- Вводится URL API сервера (по умолчанию `http://localhost:8000`)
- Кнопка "Connect" для подключения
- Статус соединения отображается в реальном времени

#### 2. **Строка статуса**
Отображает 6 ключевых показателей:
- 🤖 **System Status** - состояние системы (ONLINE/OFFLINE)
- 🔋 **Battery** - уровень батареи в %
- ⬆️ **Altitude** - текущая высота полета
- 📈 **Speed** - скорость полета
- 📡 **Signal** - качество сигнала
- ⏱️ **Flight Time** - время в полете

#### 3. **Раздел телеметрии**
Показывает в реальном времени:
- Позицию дрона (X, Y, Z координаты)
- Углы ориентации (Roll, Pitch, Yaw)

#### 4. **Управление полетом**
Быстрые кнопки команд:
- 🛫 **Takeoff** - взлет
- 🛬 **Land** - посадка
- 🏠 **RTL** - возврат на базу
- ⏸️ **Hover** - зависание

Навигация на точку:
- Введите координаты X, Y, Z
- Нажмите "Navigate"

#### 5. **Визуализация полета**
Интерактивная карта, показывающая:
- 📍 Текущую позицию дрона
- 🏠 Базу (точка взлета)
- Траекторию полета

#### 6. **Готовые сценарии**
Нажмите на карточку сценария для запуска:
- 👁️ **Patrol** - Патрулирование
- 🔍 **Search & Detect** - Поиск объектов
- 📸 **Survey** - Съемка территории
- 📦 **Delivery** - Доставка груза
- 🐝 **Swarm Flight** - Роевой полет
- 🔧 **Inspection** - Инспекция

#### 7. **Системные инструменты**
Показывает статус всех активных инструментов:
- 🟢 Активен
- 🔴 Неактивен
- 🟠 В режиме ожидания (мигает)

#### 8. **Системный лог**
Логирует все события в реальном времени:
- 🔵 INFO - информационные сообщения
- 🟢 SUCCESS - успешно выполненные действия
- 🔴 ERROR - ошибки
- 🟠 WARNING - предупреждения

## 🗺️ Управление миссиями

### 4. Создание и управление миссиями

#### 4.1 Пошаговая инструкция

1. Откройте дашборд → вкладка «Миссии».
2. Введите название миссии (например, «Патрулирование склада»).
3. Выберите тип миссии (патрулирование, обследование, доставка, поиск, свободный полёт).
4. Нарисуйте маршрут на карте:
   - Левый клик — добавить точку.
   - Правый клик по маркеру — удалить.
   - Выделите точку — задайте высоту.
5. Настройте параметры полёта (высота, скорость, действия в точках).
6. Добавьте особые инструкции текстом (например, «избегать жилых домов»).
7. Проверьте маршрут (кнопка «Предпросмотр»).
8. Сохраните миссию → она появится в списке.
9. Запустите миссию → дрон автоматически взлетит и выполнит задание.
10. Наблюдайте на вкладке «Телеметрия» (позиция, заряд, видео). В любой момент можно приостановить или вернуть дрон.

**Пример: поиск человека на открытой территории**
1. Название: «Поиск_парк_2026».
2. Тип: «Поиск».
3. На карте отметить прямоугольную область → система построит маршрут «змейкой».
4. Настройки: высота 40 м, скорость 5 м/с, действие в точках: «активировать поиск людей, при обнаружении — зависнуть и сделать 5 фото».
5. Дополнительные указания: «При обнаружении человека — отправить координаты оператору и продолжить поиск».
6. Запуск → дрон сканирует территорию, при обнаружении человека делает фото, отправляет данные и продолжает.

#### 4.2 Типы миссий

1. **Патрулирование**
   - Назначение: регулярный обход заданной зоны.
   - Особенности: замкнутый маршрут, действия в точках, автоматическое реагирование на аномалии.

2. **Поиск и обнаружение**
   - Назначение: активное сканирование территории для нахождения целей (люди, транспорт, объекты).
   - Особенности: маршрут «змейкой» или спиралью, ИИ‑анализ видео, фиксация координат.

3. **Доставка груза**
   - Назначение: транспортировка груза из точки A в точку B.
   - Особенности: проверка веса, контроль баланса, резервная процедура при потере связи.

4. **Обследование объекта**
   - Назначение: детальный осмотр зданий, сооружений.
   - Особенности: облёт по периметру или спирали, автоматическое выявление дефектов, сравнение с предыдущими инспекциями.

5. **Картографирование**
   - Назначение: создание карт, 3D‑моделей рельефа.
   - Особенности: полёт по сетке с перекрытием кадров, построение ортофотоплана, экспорт в GIS‑форматы.

6. **Сопровождение движущейся цели**
   - Назначение: автоматическое удержание цели в кадре.
   - Особенности: предсказание траектории, адаптация к скорости, передача сопровождения между дронами.

7. **Экстренное реагирование**
   - Назначение: быстрое развёртывание при ЧС (пожар, ДТП, наводнение).
   - Особенности: приоритет скорости, автономные действия (сброс аптечки, громкоговоритель), интеграция со службами.

8. **Тестовая миссия**
   - Назначение: обучение и отладка.
   - Особенности: режим симуляции, замедление времени, подсказки, запись сессии.

## 💡 Советы и рекомендации

### Лучшие практики

1. **Перед полетом:**
   - Проверьте подключение API
   - Убедитесь в статусе батареи >= 80%
   - Проверьте сигнал GPS (должен быть зеленым)

2. **Во время полета:**
   - Следите за батареей в реальном времени
   - Установите высоту полета в соответствии с местностью
   - Используйте готовые сценарии для безопасности

3. **После полета:**
   - Приземлитесь полностью
   - Дайте системе остыть
   - Сохраните логи полета (находятся в `data/flight_data/`)

### Частые вопросы

**Q: Дашборд не подключается к API**
A: Убедитесь, что:
- API сервер запущен (`python main.py`)
- Правильно введен URL
- Нет блокировки на уровне firewall

**Q: Телеметрия не обновляется**
A: 
- Проверьте подключение к API
- Перезагрузите страницу/приложение
- Проверьте логи сервера

**Q: Команда не выполняется**
A:
- Убедитесь, что дрон в состоянии готовности
- Проверьте батарею
- Посмотрите статус в системном логе

### Оптимизация производительности

- **HTML интерфейс:**
  - Используется WebSocket для реал-тайм данных
  - Локальное кэширование телеметрии
  - Оптимизированные анимации

- **Streamlit приложение:**
  - Кэширование API запросов
  - Автоматическое обновление каждые 5 секунд
  - Минимизированные перерисовки
