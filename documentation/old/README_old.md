## Введение

**COBA AI Drone Agent** — это полнофункциональная система автономного управления дронами с современным веб-интерфейсом. Система объединяет:

- **Двухуровневый ИИ**: Core Agent для принятия решений + Sub-Agent для анализа данных
- **Три системы навигации**: SLAM, EMEU, REPTOP для работы без GPS
- **Mesh-сеть**: Самоорганизующаяся сеть дронов
- **Обучение с подкреплением**: DQN для адаптивного поведения
- **Современный дашборд**: React/TypeScript интерфейс с 11 вкладками
- **ИИ-помощник**: Плавающий чат с контекстом миссий

### Ключевые возможности

- 🚁 **Управление флотом дронов** с визуализацией mesh-топологии
- 🎯 **Создание миссий** на интерактивной карте с шаблонами
- 🧠 **Обучение ИИ** в реальном времени с графиками прогресса
- 🔧 **13 инструментов** для различных задач (детекция, навигация, логистика)
- 📊 **Реал-тайм телеметрия** с графиками и картой
- 💬 **ИИ-чат** с быстрыми ответами и контекстом
- 🔒 **Безопасность** с геозонами и протоколами аварий
- 📦 **Бэкапы и отчеты** с экспортом данных

## Архитектура системы

### Бэкенд (Python/FastAPI)

- **Core Agent**: Принимает решения, управляет дроном
- **Sub-Agent**: Анализирует сенсорные данные
- **Инструменты**: 13 модулей (ObjectDetection, SLAM, EMEU и др.)
- **Mesh-сеть**: Координация флота дронов
- **Обучение**: DQN с симуляторами (AirSim, Gazebo, PyBullet)
- **API**: 50+ эндпоинтов для всех функций

### Фронтенд (React/TypeScript)

- **Фреймворк**: TanStack Start + TanStack Router
- **UI**: Tailwind CSS v4 + Radix UI (45+ компонентов)
- **Карты**: Leaflet с 4 провайдерами (OSM, Satellite, Topography, Dark)
- **Графики**: Recharts для телеметрии и обучения
- **Чат**: ИИ-помощник с smart-reply

### Связь

- **API**: RESTful API на FastAPI (порт 8000)
- **WebSocket**: Реал-тайм данные (опционально)
- **Протоколы**: MAVLink, собственный mesh-протокол

## Вкладки дашборда

### 1. Телеметрия (главная)

**Функции:**
- Графики высоты, скорости, батареи, температуры, сигнала
- Выбор активного дрона из списка
- Карта с положением дронов и маршрутами миссий
- Кнопки управления: Пауза, RTL, ARM/DISARM, Старт миссии
- Автообновление телеметрии (1-10 сек)

**API**: `GET /api/v1/telemetry`

### 2. Миссии

**Функции:**
- Создание миссий кликом на карте
- Шаблоны: патруль, поиск, доставка, обследование
- Планирование маршрута с препятствиями
- Очереди миссий, расписания, условный запуск
- Импорт/экспорт: JSON, KML, GPX, CSV

**API**: `POST /api/v1/mission/start`, `GET /api/v1/missions`

### 3. Инструкции

**Функции:**
- Добавление/редактирование правил и законов
- Геозоны безопасности
- Протоколы аварийного реагирования
- Связка с миссиями и инструментами

**API**: `GET /api/v1/instructions`

### 4. Инструменты

**Функции:**
- 13 модулей COBA: ObjectDetection, SLAM, EMEU, REPTOP, etc.
- Карточки с описанием и переключателями ВКЛ/ВЫКЛ
- Панель настроек для каждого инструмента
- Тестирование инструментов без реального воздействия

**API**: `GET /api/v1/tools`, `POST /api/v1/tools/{tool}/execute`

### 5. Обучение

**Функции:**
- Выбор симулятора: AirSim, Gazebo, PyBullet, SkyRover
- Обучение DQN с графиками reward/loss
- Анализ результатов с текстовыми рекомендациями
- Экспорт обученных моделей

**API**: `POST /api/v1/learning/start`, `GET /api/v1/learning/progress`

### 6. Флот

**Функции:**
- Список дронов с статусом (online/offline/mission/charging)
- Bulk-команды для группы дронов
- Визуализация mesh-топологии
- Формации и смена лидера

**API**: `GET /api/v1/fleet/status`, `POST /api/v1/fleet/formation`

### 7. Камера

**Функции:**
- Видеострим с дрона
- Детекция объектов с аннотациями
- Запись видео
- Масштабирование и настройки яркости

**API**: `GET /api/v1/camera/stream`, `POST /api/v1/camera/record/start`

### 8. События

**Функции:**
- Журнал событий с фильтрами по уровню/источнику/времени
- График активности
- Экспорт в CSV/JSON
- Настройка уведомлений (email, Telegram, webhooks)

**API**: `GET /api/v1/events/log`, `POST /api/v1/events/alert/config`

### 9. Бэкапы

**Функции:**
- Создание бэкапов компонентов (миссии, конфиг, модели)
- Восстановление с выбором режима (merge/replace)
- История бэкапов с размерами и датами

**API**: `POST /api/v1/backup/create`, `GET /api/v1/backup/list`

### 10. Настройки

**Функции:**
- Визуальные сенсоры: камера, тепловизор, LiDAR
- Сенсоры связи: Wi-Fi, LoRa, mesh
- IMU и калибровки
- Тестирование всех систем

**API**: `GET /api/v1/settings`, `POST /api/v1/settings`

### 11. Симуляция

**Функции:**
- Переключение между симуляторами
- Конфигурация окружения (погода, местность)
- Управление виртуальным флотом

**API**: `POST /api/v1/simulation/start`

## Устранение неполадок

### Проблема: Фронтенд не запускается
**Решение:**
```bash
rm -rf node_modules package-lock.json
npm install --legacy-peer-deps
npm run dev
```

### Проблема: API недоступен
**Решение:**
```bash
python main.py api
curl http://localhost:8000/health
```

### Проблема: Карта не загружается
**Решение:** Проверить интернет-соединение и выбрать другой провайдер карт в настройках.

### Проблема: Обучение не запускается
**Решение:** Установить зависимости симулятора или использовать mock-режим.

- **Бэкенд**: `logs/agent.log`
- **Фронтенд**: Консоль браузера

**Цикл работы:**
1. **Воспринимает** — собирает данные с датчиков
2. **Анализирует** — проверяет опасности и соответствие плану
3. **Принимает решение** — выбирает следующее действие
4. **Действует** — отправляет команды исполнительным устройствам
5. **Проверяет результат** — убеждается в выполнении

**Аварийные протоколы (автоматические):**

| Ситуация | Действие дрона |
|----------|----------------|
| Заряд < 20% | Возврат в точку старта и посадка |
| Потеря сигнала (>5 сек) | Автономная навигация, возврат по последнему маршруту |
| Перегрев моторов (>60°C) | Немедленная посадка |
| Потеря позиционирования (>10 сек) | Зависание, сканирование окружения |
| Отказ мотора | Перераспределение тяги, аварийная посадка |
| Выход за разрешённую зону | Остановка, возврат к границе |

### 2.2 Sub-Agent — помощник-аналитик

**Функции:**
- Сбор и анализ данных с камер, датчиков, навигационных систем
- Ведение журналов полёта
- Построение карт местности в реальном времени
- Расчёт оптимальных маршрутов
- Предоставление рекомендаций Core Agent (без права самостоятельного управления)

**Пример взаимодействия:**
Sub-Agent обнаруживает движение на видео → сообщает Core Agent: «Рекомендую снизиться и сделать круг над точкой». Core Agent принимает решение: согласиться, отклонить или выбрать альтернативу.

## Веб-дашборд

Веб-интерфейс на **React + Vite** (`npm run dev`) по умолчанию открывается на `http://localhost:5173` и ходит к каноническому API **v4** (`src/api/rest_api.py`) на `http://localhost:8000`. Отдельный Streamlit-дашборд (`dashboard/app.py`) может использоваться параллельно — см. режимы запуска в `main.py`.

### Запуск стека React + API v4

**Бэкенд** (агент + REST/WebSocket):

```bash
pip install -r requirements.txt
python main.py api
```

Сервис: `http://localhost:8000`, Swagger: `http://localhost:8000/docs`, WebSocket телеметрии: `ws://localhost:8000/ws/telemetry`.

**Фронтенд**:

```bash
npm install
npm run dev
npm run typecheck
npm run lint
```

Скопируйте `.env.example` в `.env` при необходимости (`VITE_API_BASE_URL`, `VITE_WS_URL`). В режиме разработки Vite проксирует `/api`, `/health` и `/ws` на порт 8000 (см. `vite.config.ts`), можно оставить `VITE_API_BASE_URL` пустым и вызывать относительные пути.

**Автотесты API v4 и Core Agent**:

```bash
python -m pytest src/tests/test_rest_api_v4.py src/tests/test_core_agent.py -q
```

### Ручной чек-лист UI (демо-fallback без дрона)

1. Открыть `http://localhost:5173` — индикатор «API доступен» в шапке при работающем `python main.py api`.
2. **Телеметрия**: значения обновляются; подсказки «?» открываются по клику.
3. **Команды**: GOTO с координатами шлёт `POST /api/v1/command`; аварийные кнопки — `/api/v1/emergency/land` и `/api/v1/emergency/stop`.
4. **Миссии**: «Запустить» → активная миссия в статусе `running`; «Остановить» сбрасывает активную; «Сохранить» пишет JSON в `data/missions/`.
5. **Инструменты**: список из `/api/v1/tools`; «Запустить» — `/api/v1/tools/execute`.
6. **Обучение**: кнопки вызывают `/api/v1/learning/*`; при `experimental: true` видна плашка демо.
7. **Флот**: формации и средний заряд из `/api/v1/fleet/status`.
8. **Камера**: снимок `/api/v1/camera/snapshot`; слайдеры — zoom/brightness; запись — record/start|stop.
9. **События**: `/api/v1/events` и статистика.
10. **Бэкапы**: создание и список; скачивание по ссылке на `/api/v1/backup/download/{id}`.
11. **Чат**: `POST /api/v1/sub_agent/ask` (офлайн-ответ без ключа DeepSeek).

## Сенсоры

### Навигационные инструменты

#### 1. SlamNavigation
- **Принцип**: камера фиксирует устойчивые ориентиры, сравнивает кадры и вычисляет перемещение
- **Точность**: ±0,5 м в помещении, ±2 м на открытой местности
- **Применение**: внутри зданий, в лесу, в городских каньонах

#### 2. EMEU
- **Принцип**: фильтр Калмана пересчитывает положение на основе ускорений и поворотов
- **Особенность**: полностью автономна, ошибка накапливается ~1 м/мин
- **Применение**: резерв при потере всех внешних сигналов

#### 3. REPTOP
- **TERCOM**: сравнение рельефа с цифровой картой
- **Звёздная навигация**: определение ориентации по звёздам
- **Магнитометрическая**: сравнение аномалий магнитного поля
- **Точность**: ±10–50 м (требует предварительной загрузки карт)

## Практические сценарии

### Сценарий 1: Поиск человека в лесу

**Подготовка:**
- Проверить заряд (100%), камеру и тепловизор
- Загрузить актуальные карты
- Создать миссию «Поиск и обнаружение»

**Выполнение:**
1. Запустить миссию
2. Дрон автоматически набирает высоту и начинает облёт
3. При обнаружении цели → получить уведомление
4. Подтвердить и передать координаты спасателям

### Сценарий 2: Инспекция промышленного объекта

**Подготовка:**
- Загрузить планы объекта
- Выбрать сенсоры: камера, тепловизор, лидар
- Создать миссию «Обследование объекта»

**Выполнение:**
1. Дрон облетает объект по заданному маршруту
2. Система автоматически выделяет дефекты
3. Сравнивает с предыдущими инспекциями
4. Генерирует отчёт в PDF или интерактивной карте
